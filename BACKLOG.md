# ClaudeBar — бэклог

Рабочий список развития. Не roadmap с датами, а очередь идей с оценкой.

## Сделано (v0.1)

- Панель всегда-поверх со списком окон VS Code / Cursor, группировка по имени проекта.
- ЛКМ — переключиться на окно; ПКМ — цвет (8) и произвольная метка.
- Привязка цвета/метки по имени проекта, конфиг `claudebar.ini`, перетаскивание, авто-обновление ~1с.
- Нативный exe ~280 КБ (Rust + Win32), опубликован: https://github.com/Baho73/claudebar
- Проверен на Kaspersky OpenTIP — вердикт Clean (0 детектов).

## Большие фичи

### A. Мультиприложения (не только Claude/редакторы)
Поддержать Word, Excel, MS Project и любые приложения, у которых «по куче окон без вкладок».
- Перейти с фильтра «по суффиксу заголовка» на определение приложения **по имени процесса** (`Code.exe`, `Cursor.exe`, `WINWORD.exe`, `EXCEL.EXE`, `WINPROJ.exe`, …).
- Конфиг: список `app=<процесс>|<имя блока>|<правило извлечения имени документа из заголовка>`.
- Технически: к каждому окну добрать процесс через `GetWindowThreadProcessId` + `QueryFullProcessImageNameW`.
- Сложность: средняя.

### B. Блоки-секции со сворачиванием (крыжик)
Группировать окна по приложению в секции с заголовком; каждая секция сворачивается треугольником, состояние — в конфиг.
- Зависит от A (нужно знать приложение каждого окна).
- Сложность: низкая–средняя.

### C. «Недавние / закрытые документы» под катом в каждом блоке
Под активными окнами блока — сворачиваемый список недавних документов этого приложения; клик открывает документ. Когда окно закрывается, документ «уезжает» в этот список.
- Открытие документа: `ShellExecuteW(path)`.
- Отслеживание «закрылось → под кат»: диффить список окон между опросами; пропавшее окно → его документ в «недавние».
- **Открытый вопрос — откуда брать список недавних (с путями!):**
  1. Своё отслеживание — запоминать открытые документы. Минус: из заголовка Word/Project нет полного пути, открыть потом нечем.
  2. **Windows Recent** — `%APPDATA%\Microsoft\Windows\Recent\*.lnk`, парсить ярлыки в пути, фильтровать по расширению (`.docx`→Word, `.xlsx`→Excel, `.mpp`→Project). Плюс: полные пути, открытие тривиально.
  3. **Office MRU** — реестр `HKCU\Software\Microsoft\Office\<версия>\Word\User MRU\…\File MRU`. Плюс: точно по приложению. Минус: путь зависит от версии Office, свой формат записи.
- Сложность: высокая (отдельный модуль: источник данных + парсинг + сопоставление окно↔файл).

## Архитектурная заметка

Идеи A+B+C превращают утилиту из «переключателя» в менеджер окон и документов. `main.rs` уже ~750 строк; при таком росте стоит разбить на модули (`win_enum`, `config`, `render`, `recent_docs`). На этом этапе уместно поднять GRACE для управления растущим проектом.
