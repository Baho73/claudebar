# «Где я это обсуждал» — поиск по чатам и файлам с подсветкой в ClaudeBar

Статус: утверждён. clfind MVP реализован и принят (индекс построен: 29795 чанков, 77 папок).
v2-архитектура: BM25 нативно в ClaudeBar (rusqlite), Python+FAISS = dense-fallback (только при пустом BM25).
Track 2 (ClaudeBar `M-SEARCH`) → grace-plan. Даты: 2026-06-20 дизайн, 2026-06-21 v2.

## 1. Проблема и цель

Пользователь постоянно упирается в вопрос «где я это обсуждал». Нужно найти **проект (папку)**, в чьих чатах с ИИ (или, по кнопке, в файлах) встречалось искомое. Иногда известны точные слова (например telegram-ник человека), иногда только смысл — поэтому нужны **оба** типа поиска: лексический (точные слова) и семантический (по смыслу).

**Результат поиска — подсветка папки проекта в ClaudeBar, а не выдача текста.** Содержимое не показываем. Перемотка к точному файлу/месту внутри папки — приятный бонус по возможности, не обязательна.

## 2. Не-цели (YAGNI)

- Не показываем сниппеты/содержимое в панели — только подсветка папки.
- Источник чатов только Claude Code транскрипты (не Telegram/ChatGPT/прочее).
- Не индексируем `.mpp` (бинарь MS Project), медиа, бинарники.
- Перемотка к точному месту — best-effort, отдельной фазой.
- Не RAG/не генерация ответов — только поиск и подсветка.

## 3. Архитектура (v2): нативный BM25 + Python-FAISS как fallback

```
ClaudeBar (Rust)                          clfind (Python)
  окошко поиска                           ─ строит индекс (clfind index, GPU)
  ├─ BM25 нативно ─► rusqlite ─► clfind.db (FTS5, WAL) ◄─ пишет indexer
  │    живой, с 3-й буквы, БЕЗ Python
  └─ если BM25 пусто ─ HTTP ─► демон /search(dense) ─► FAISS + модель (CPU)
       спавнит демон при необходимости                ◄─ читает clfind.faiss
```

1. **clfind** (Python) — СТРОИТ индекс (парсинг/чанкинг/эмбеддинг → `clfind.db` FTS5 + `clfind.faiss`) и поднимает HTTP-демон ТОЛЬКО для семантики (dense, llama.cpp).
2. **ClaudeBar** (Rust/Win32, GRACE, модуль `M-SEARCH`) — читает `clfind.db` напрямую (`rusqlite`) и гонит **BM25 нативно** (живой поиск, без Python). Если BM25 ничего не нашёл — спавнит/греет демон и просит dense. Подсветка папок.

**Ключевой принцип:** Python+FAISS поднимается только когда по словам пусто — экономия (помнишь слова → Python не трогаем).

## 4. Компонент 1: clfind (Python)

### 4.1 Источники и нормализация
- **Чаты:** `~/.claude/projects/<enc>/*.jsonl`. Парсер берёт строки `type=user|assistant`, поле `message.content` (строка или список блоков `type=text`). Отсев: `tool_use`/`tool_result`, `attachment`, `file-history-snapshot`, base64, служебные (`mode`, `permission-mode`, `last-prompt`).
- **Папка проекта берётся из поля `cwd`** в строках транскрипта (подтверждено: `"cwd":"D:\\Python\\claudebar"`). Это надёжнее декодирования имени каталога (`D--Python-hh-answer`), где дефис неоднозначен.
- **Файлы** (scope=all): корни проектов из конфига (напр. `D:\Python`, опц. Obsidian/YandexDisk). Текст/код по расширению (`.py .rs .md .txt .json .toml .yaml .js .ts .html .css ...`); `.docx` (python-docx), `.xlsx` (openpyxl), `.pdf` (pypdf). Исключаем `.git`, `node_modules`, `target`, `__pycache__`, медиа/бинарь, `.mpp`.

### 4.2 Чанкинг
- **Чаты:** по реплике (одно user/assistant сообщение); длинные режем окном ~512 токенов с перекрытием ~64. Метаданные: `{project_folder, source=chat, session_id, ts, role, text}`.
- **Файлы:** скользящее окно ~512/overlap 64 по тексту. Метаданные: `{project_folder, source=file, path, line_start, text}`.

### 4.3 Хранилище (всё файловое, как просил пользователь)
- **SQLite** (`clfind.db`): таблица `chunks(id, project_folder, source, path|session, location, text, src_mtime)`; виртуальная **FTS5**-таблица над `text` (ранкинг BM25).
- **FAISS** (`clfind.faiss`): dense-векторы чанков, позиция ↔ `chunks.id` через `IndexIDMap2`. Метрика cosine (`IndexFlatIP` на L2-нормированных; при росте корпуса — `IVF`).
- **Манифест** проиндексированных источников (`path|session → mtime`) для инкремента.

### 4.4 Модель эмбеддингов
- **Qwen3-Embedding-4B** (по умолчанию), `model` — строка в конфиге (sentence-transformers/transformers грузит по имени; переключение на bge-m3 / Qwen3-0.6B бесплатно).
- Query/document instruction-prefix по требованиям Qwen3; L2-нормализация.
- **Резидентность (ленивая, prefetch по первой клавише):** модель **не висит в RAM** между поисками (4B это ~12–25% от 64ГБ — держать постоянно жирно). ClaudeBar при **первом нажатии клавиши** в окошке шлёт демону `POST /warmup` → демон асинхронно грузит модель в RAM, пока пользователь дописывает запрос (загрузка ~10–30с прячется за временем набора). После поиска и короткого простоя (`idle_evict_sec`) модель выгружается, RAM освобождается.
- **Устройства:** **GPU трогает только часовой индексатор** (батч-эмбеддинг). **Запрос энкодится на CPU** (forward pass 1–2 слов ~1с). Та же модель обязательна и для индекса, и для запроса (иначе векторы несравнимы).
- **Graceful degradation:** BM25 модели не требует и отвечает мгновенно на Enter (точные слова → сразу); dense-результаты до-приезжают, как только модель догрузилась и посчитала вектор запроса. Семантика «когда слов не помню» доступна всегда, ценой ожидания загрузки (сокращённого prefetch-ом).

### 4.5 Индексация (инкрементальная, GPU-вежливая)
- CLI: `clfind index [--scope chats|all] [--reindex]`.
- Инкремент по `mtime`: новые/изменённые источники переэмбеддятся и `add` в FAISS + upsert в SQLite; неизменённое не трогаем; удалённое чистим (`remove_ids` или периодический rebuild).
- **Планировщик «дообучивания»:** раз в час (Windows Task Scheduler или внутренний таймер демона) проверяет загрузку GPU (`pynvml`/`nvidia-smi`: utilization + free VRAM). Карты свободны → индексирует появившееся за час. Карты заняты → пропуск цикла, ждёт следующего. (Планировщик и GPU-проверка — наш код; FAISS делает только `add`/`search`/persist.)
- Первый полный прогон чатов (~1.3 ГБ) на Qwen3-4B/3060 ≈ 1–2 ч (опц. распараллелить на 2 карты); далее инкремент быстрый.

### 4.6 Поиск (HTTP-демон)
> **v2:** BM25 ClaudeBar делает нативно (`rusqlite` поверх `clfind.db`), демон нужен только для **dense**. Эндпоинт `mode=bm25` остаётся (для curl/отладки), но боевой путь BM25 — нативный. `/warmup` поглощается dense-fallback (демон сам грузит модель при `mode=dense`).
- Локальный HTTP (FastAPI/uvicorn) на `127.0.0.1:<port>` из конфига. Эндпоинты:
  - `POST /warmup` — начать асинхронную загрузку модели в RAM (шлётся по первой клавише). Идемпотентно.
  - `POST /search {q, scope=chats|all, mode=bm25|full, limit}` — поиск. `mode=bm25` мгновенный, без модели (для живого поиска по мере набора); `mode=full` добавляет dense (использует/ждёт модель).
  - `GET /health` — живость + статус модели (`cold|loading|ready`).
- Конвейер запроса (две стадии, graceful):
  1. **BM25 (`mode=bm25` и `full`; мгновенно, без модели):** FTS5 `MATCH` (последний токен как префикс `term*`) → топ-N чанков со скором. Для живого поиска возвращается сразу.
  2. **Dense (только `mode=full`, когда модель готова):** энкод `q` на **CPU** → FAISS search → топ-N. Если модель ещё грузится — ждём её (prefetch по первой клавише сократил ожидание) либо отдаём dense вторым ответом.
  3. **Без кросс-слияния — раздельные сигналы.** Каждая модальность ранжируется сама по себе (BM25-скор; cosine для отбора топ-чанков). Папка помечается, какими модальностями найдена.
  4. **Агрегация по `project_folder`:** для каждой папки — отдельные скоры/флаги `bm25` и `dense`, `best` = лучший чанк (`source, path, location`).
  5. Ответ: `{results:[{folder, bm25:<score|null>, dense:<score|null>, best:{source,path,location}}], model:"ready|loading"}`. `snippet` движок знает, но в ответ для UI не кладём (нужен только для будущего jump).
- MVP может отдавать единый ответ (BM25 сразу, ждём модель для dense); стрим «BM25-сразу, dense-довеском» — необязательная оптимизация Фазы 2.

### 4.7 Конфиг clfind (TOML)
`model`, `port`, `roots[]` (для file-scope), `index_device=cuda`, `query_device=cpu`, `batch_size`, `idle_evict_sec`, `index_interval=3600`, `gpu_free_vram_min`, `live_min_chars=3`.

## 5. Компонент 2: ClaudeBar (Rust) — модуль M-SEARCH

- **Окошко поиска в шапке** (Win32 `EDIT`), показ по глифу 🔍 или горячей клавише. Поле + переключатель scope `[Чаты | +Файлы]` (по умолчанию Чаты).
- **Живой BM25 нативно с 3-го символа:** на каждое нажатие (≥`live_min_chars`=3, debounce ~120мс) ClaudeBar через `rusqlite` открывает общий `clfind.db` и сам гонит FTS5 `MATCH` (последний токен как префикс `telegr*`, токенайзер `unicode61`), агрегирует по папке → **подсвечивает жёлтым**, на лету сужая. БЕЗ Python, БЕЗ HTTP — работает даже если демон не запущен.
- **Fallback на смысл (только если BM25 пусто):** если BM25 не дал ни одной папки — ClaudeBar поднимает Python-демон (спавн `clfind serve`, если не запущен), шлёт `/search {mode:dense}` в **отдельном потоке** (UI не морозим), ждёт модель и подсвечивает найденное **синим**. Так модель грузится ровно в случае «слов не помню».
- **Цвет:** 🟡 жёлтый = нашли по точным словам (BM25); 🔵 синий = по словам пусто → нашли по смыслу (dense). **Зелёного нет** (dense запускается только при пустом BM25, пересечения модальностей не бывает). Цвет — левой полосой строки (как «звоночек»).
- **Подсветка / выдача:**
  - Открытые окна с этим проектом → подсветка строки (полоса нужного цвета).
  - Закрытые совпадения → блок **«Найдено ещё»** (как «недавние»): строка с именем папки и цветовой полосой, клик = открыть папку/редактор (`ShellExecute` или `code <folder>`).
- **Сброс:** Esc / очистка поля → снять подсветку и блок.
- **Бонус jump:** если `best.path` и `source=file` → клик открывает файл (`ShellExecute`); `source=chat` → максимум открыть проект. Деталь отложена (Фаза 4).
- **Кнопка «Запустить поиск»:** если демон нужен (BM25 пуст) и `/health` молчит — спавним его автоматически; плюс явный пункт меню/кнопка для ручного запуска.

## 6. Запуск и IPC
- **Общий файл `clfind.db`** (SQLite, режим **WAL**): indexer (Python) пишет, ClaudeBar (Rust/`rusqlite`) читает конкурентно. BM25 — нативно из этого файла, без IPC.
- **Демон (Python, dense) поднимается лениво:** ClaudeBar спавнит `clfind serve`, когда BM25 пуст и нужна семантика (а также кнопкой «Запустить поиск» и/или автозагрузкой). Команда запуска — в `claudebar.ini` (`search_daemon_cmd=`, напр. `D:\Python\clfind\.venv\Scripts\pythonw.exe -m clfind.cli serve`).
- **ClaudeBar ↔ демон:** HTTP `127.0.0.1:<port>` только для dense (`/search {mode:dense}`, `/health`). После спавна ClaudeBar поллит `/health` до `ready`.

## 7. Тестирование
- **clfind** (pytest, фикстуры): парсинг jsonl (отсев шума, извлечение `cwd` и текста), чанкинг, тегирование папок по модальностям (bm25/dense), агрегация по папке. Качество поиска — ручная проверка на реальных запросах (вкл. кейс «telegram-ник»).
- **ClaudeBar** (cargo test, GRACE-разметка): чистые функции — парсинг JSON-ответа → папки с флагами bm25/dense; **выбор цвета (жёлтый/синий/зелёный)** по флагам; маппинг папка→секция/строка; решение «открытая/закрытая». UI/HTTP — ручной smoke.

## 8. Фазировка (MVP → дальше)
- **Фаза 1 (MVP):** clfind индекс чатов (FTS5 + FAISS + Qwen3-4B) + демон `/search` + инкремент по `mtime` (ручной `clfind index` + простой таймер); ClaudeBar окошко + подсветка открытых + блок закрытых с открытием папки. scope=chats.
- **Фаза 2:** GPU-вежливый планировщик «раз в час, если карты свободны» (`pynvml`-гейт); scope=all (код/текст в корнях проектов), кнопка +Файлы.
- **Фаза 3:** Office/PDF извлечение (docx/xlsx/pdf).
- **Фаза 4 (бонус):** jump-to-file/место (открыть файл по `best.path`, для чата — открыть проект).

## 9. Проверить на реализации
- Точный формат вызова Qwen3-Embedding-4B (query vs document instruction, нормализация; sentence-transformers vs FlagEmbedding vs transformers).
- `faiss` на Windows: `faiss-cpu` (есть wheels) для search; encode на CUDA через torch. (`faiss-gpu` на Windows проблемный; search на CPU для personal-scale быстр.)
- Финальная сверка модели по свежему ruMTEB перед стартом (кандидаты: Qwen3-Embedding-4B/8B, GigaEmbeddings, bge-m3).
- Имя проекта clfind (переименуемо).

## 10. Зафиксированные решения (из брейншторма)
- Источник: только Claude Code транскрипты; папка из `cwd`.
- Эмбеддинги: локально на CUDA; модель Qwen3-Embedding-4B (конфиг-переключатель).
- **Поиск (v2): BM25 нативно в ClaudeBar** (`rusqlite` поверх общего `clfind.db` FTS5, WAL) — живой, с 3-го символа, без Python. **Python+FAISS (dense) — fallback, поднимается ТОЛЬКО когда BM25 пуст.** Цвет: 🟡 BM25, 🔵 dense (зелёного нет).
- Связь ClaudeBar↔демон: HTTP localhost, только dense. Демон спавнится ClaudeBar при необходимости (+ кнопка/автозагрузка), команда в `claudebar.ini` (`search_daemon_cmd=`).
- Резидентность: модель не висит в RAM; грузится при dense-fallback, выгружается после простоя. GPU — только индексатор; dense-запрос на CPU.
- Охват чатов: только основные сессии (`projects/<proj>/<session>.jsonl`); субагентские транскрипты (глубже) исключены.
- Бэкенд модели: llama.cpp (GGUF Qwen3-Embedding-4B Q5_K_M), без torch. Индекс построен и принят (29795 чанков, 77 папок).
- Единица результата: папка проекта; подсветка, не выдача текста; закрытые — показать и дать открыть.
- Охват файлов: корни проектов + код/текст + docx/xlsx/pdf (без .mpp).
- GPU: 2× RTX 3060 12 ГБ.
