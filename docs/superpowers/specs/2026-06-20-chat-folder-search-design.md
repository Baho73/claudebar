# «Где я это обсуждал» — поиск по чатам и файлам с подсветкой в ClaudeBar

Статус: дизайн (брейншторм завершён, ждёт ревью). Дата: 2026-06-20.

## 1. Проблема и цель

Пользователь постоянно упирается в вопрос «где я это обсуждал». Нужно найти **проект (папку)**, в чьих чатах с ИИ (или, по кнопке, в файлах) встречалось искомое. Иногда известны точные слова (например telegram-ник человека), иногда только смысл — поэтому нужны **оба** типа поиска: лексический (точные слова) и семантический (по смыслу).

**Результат поиска — подсветка папки проекта в ClaudeBar, а не выдача текста.** Содержимое не показываем. Перемотка к точному файлу/месту внутри папки — приятный бонус по возможности, не обязательна.

## 2. Не-цели (YAGNI)

- Не показываем сниппеты/содержимое в панели — только подсветка папки.
- Источник чатов только Claude Code транскрипты (не Telegram/ChatGPT/прочее).
- Не индексируем `.mpp` (бинарь MS Project), медиа, бинарники.
- Перемотка к точному месту — best-effort, отдельной фазой.
- Не RAG/не генерация ответов — только поиск и подсветка.

## 3. Архитектура: два компонента

```
                 HTTP localhost
ClaudeBar (Rust) ───────────────▶ clfind (Python демон)
  окошко поиска                     /search  → BM25(FTS5)+dense(FAISS), RRF
  подсветка папок ◀─── JSON ───     /health
                  [{folder,score,                │
                    best:{path,loc}}]            │ читает
                                                 ▼
                          ~/.claude/projects/*/*.jsonl  (чаты)
                          корни проектов: код/текст/docx/xlsx/pdf  (файлы)
                                                 │
                          планировщик (раз в час, если GPU свободен)
                          → инкрементальный индекс: SQLite + FAISS (файлы)
```

1. **clfind** — новый Python-проект (имя рабочее, переименуемо): индекс + HTTP-демон, CUDA.
2. **ClaudeBar** (существующий, Rust/Win32, под GRACE) — окошко поиска + подсветка папок. Новый модуль `M-SEARCH`.

## 4. Компонент 1: clfind (Python)

### 4.1 Источники и нормализация
- **Чаты:** `~/.claude/projects/<enc>/*.jsonl`. Парсер берёт строки `type=user|assistant`, поле `message.content` (строка или список блоков `type=text`). Отсев: `tool_use`/`tool_result`, `attachment`, `file-history-snapshot`, base64, служебные (`mode`, `permission-mode`, `last-prompt`).
- **Папка проекта берётся из поля `cwd`** в строках транскрипта (подтверждено: `"cwd":"D:\\Python\\claudebar"`). Это надёжнее декодирования имени каталога (`D--Python-hh-answer`), где дефис неоднозначен.
- **Файлы** (scope=all): корни проектов из конфига (напр. `D:\Python`, опц. Obsidian/YandexDisk). Текст/код по расширению (`.py .rs .md .txt .json .toml .yaml .js .ts .html .css ...`); `.docx` (python-docx), `.xlsx` (openpyxl), `.pdf` (pypdf). Исключаем `.git`, `node_modules`, `target`, `__pycache__`, медиа/бинарь, `.mpp`.

### 4.2 Чанкинг
- **Чаты:** по реплике (одно user/assistant сообщение); длинные режем окном ~512 токенов с перекрытием ~64. Метаданные: `{project_folder, source=chat, session_id, ts, role, text}`.
- **Файлы:** скользящее окно ~512/overlap 64 по тексту. Метаданные: `{project_folder, source=file, path, line_start, text}`.

### 4.3 Хранилище (всё файловое, как просил пользователь)
- **SQLite** (`clfind.db`): таблица `chunks(id, project_folder, source, path|session, location, text, src_mtime)`; виртуальная **FTS5**-таблица над `text` (ранкинг BM25).
- **FAISS** (`clfind.faiss`): dense-векторы чанков, позиция ↔ `chunks.id` через `IndexIDMap2`. Метрика cosine (`IndexFlatIP` на L2-нормированных; при росте корпуса — `IVF`).
- **Манифест** проиндексированных источников (`path|session → mtime`) для инкремента.

### 4.4 Модель эмбеддингов
- **Qwen3-Embedding-4B** (по умолчанию), `model` — строка в конфиге (sentence-transformers/transformers грузит по имени; переключение на bge-m3 / Qwen3-0.6B бесплатно).
- Query/document instruction-prefix по требованиям Qwen3; L2-нормализация.
- **Резидентность (GPU-вежливость):** модель живёт в **CPU RAM**; на GPU поднимается только на время запроса/батча индексации, после простоя (idle-timeout) выгружается обратно. Так одна 3060 не занята под модель 24/7. Первый запрос после простоя +1–2с (перенос на GPU).

### 4.5 Индексация (инкрементальная, GPU-вежливая)
- CLI: `clfind index [--scope chats|all] [--reindex]`.
- Инкремент по `mtime`: новые/изменённые источники переэмбеддятся и `add` в FAISS + upsert в SQLite; неизменённое не трогаем; удалённое чистим (`remove_ids` или периодический rebuild).
- **Планировщик «дообучивания»:** раз в час (Windows Task Scheduler или внутренний таймер демона) проверяет загрузку GPU (`pynvml`/`nvidia-smi`: utilization + free VRAM). Карты свободны → индексирует появившееся за час. Карты заняты → пропуск цикла, ждёт следующего. (Планировщик и GPU-проверка — наш код; FAISS делает только `add`/`search`/persist.)
- Первый полный прогон чатов (~1.3 ГБ) на Qwen3-4B/3060 ≈ 1–2 ч (опц. распараллелить на 2 карты); далее инкремент быстрый.

### 4.6 Поиск (HTTP-демон)
- Локальный HTTP (FastAPI/uvicorn) на `127.0.0.1:<port>` из конфига. `POST /search {q, scope=chats|all, limit}`, `GET /health`.
- Конвейер запроса:
  1. **BM25:** FTS5 `MATCH` → топ-N чанков со скором.
  2. **Dense:** энкод `q` на GPU → FAISS search → топ-N.
  3. **Слияние RRF** (k=60) объединяет два ранга.
  4. **Агрегация по `project_folder`:** скор папки = по лучшим её чанкам; `best` = лучший чанк (`source, path, location`).
  5. Ответ: `{results:[{folder, score, best:{source,path,location}}]}`. `snippet` движок знает, но в ответ для UI не кладём (нужен только для будущего jump).

### 4.7 Конфиг clfind (TOML)
`model`, `port`, `roots[]` (для file-scope), `device=cuda`, `batch_size`, `idle_offload_sec`, `index_interval=3600`, `gpu_free_vram_min`.

## 5. Компонент 2: ClaudeBar (Rust) — модуль M-SEARCH

- **Окошко поиска в шапке** (Win32 `EDIT`), показ по глифу 🔍 в шапке или горячей клавише. Поле + переключатель scope `[Чаты | +Файлы]` (по умолчанию Чаты).
- **На Enter:** HTTP POST к демону в **отдельном потоке** (UI не морозим), парсим JSON → множество папок.
- **Подсветка / выдача:**
  - Открытые окна с этим проектом → подсветка строки (переиспользуем механику «звоночка», новый вид подсветки `search-hit`).
  - Закрытые совпадения → блок **«Найдено ещё»** (как «недавние»): строка с именем папки, клик = открыть папку/редактор (`ShellExecute` или `code <folder>`).
- **Сброс:** Esc / очистка поля → снять подсветку и блок.
- **Бонус jump:** если `best.path` и `source=file` → клик открывает файл (`ShellExecute`); `source=chat` → максимум открыть проект (перемотка в `.jsonl` не человекочитаема). Деталь отложена (Фаза 4).
- **Демон недоступен** (`/health` молчит) → строка статуса «движок не запущен» (опц. предложить запустить clfind).

## 6. Запуск и IPC
- Демон clfind стартует в фоне: автозагрузка (ярлык, как у ClaudeBar) или ClaudeBar спавнит при старте (config).
- ClaudeBar ↔ демон: HTTP `127.0.0.1:<port>` (просто, отлаживается curl-ом).

## 7. Тестирование
- **clfind** (pytest, фикстуры): парсинг jsonl (отсев шума, извлечение `cwd` и текста), чанкинг, RRF-слияние (детерминированный тест), агрегация по папке. Качество поиска — ручная проверка на реальных запросах (вкл. кейс «telegram-ник»).
- **ClaudeBar** (cargo test, GRACE-разметка): чистые функции — парсинг JSON-ответа → список папок; маппинг папка→секция/строка; решение «открытая/закрытая». UI/HTTP — ручной smoke.

## 8. Фазировка (MVP → дальше)
- **Фаза 1 (MVP):** clfind индекс чатов (FTS5 + FAISS + Qwen3-4B) + демон `/search` + инкремент по `mtime` (ручной `clfind index` + простой таймер); ClaudeBar окошко + подсветка открытых + блок закрытых с открытием папки. scope=chats.
- **Фаза 2:** GPU-вежливый планировщик «раз в час, если карты свободны» (`pynvml`-гейт); scope=all (код/текст в корнях проектов), кнопка +Файлы.
- **Фаза 3:** Office/PDF извлечение (docx/xlsx/pdf).
- **Фаза 4 (бонус):** jump-to-file/место (открыть файл по `best.path`, для чата — открыть проект).

## 9. Проверить на реализации
- Точный формат вызова Qwen3-Embedding-4B (query vs document instruction, нормализация; sentence-transformers vs FlagEmbedding vs transformers).
- `faiss` на Windows: `faiss-cpu` (есть wheels) для search; encode на CUDA через torch. (`faiss-gpu` на Windows проблемный; search на CPU для personal-scale быстр.)
- Финальная сверка модели по свежему ruMTEB перед стартом (кандидаты: Qwen3-Embedding-4B/8B, GigaEmbeddings, bge-m3).
- Имя проекта clfind (переименуемо).

## 10. Зафиксированные решения (из брейншторма)
- Источник: только Claude Code транскрипты; папка из `cwd`.
- Эмбеддинги: локально на CUDA; модель Qwen3-Embedding-4B (конфиг-переключатель).
- Поиск: гибрид BM25 (FTS5) + dense (FAISS), слияние RRF.
- Связь ClaudeBar↔движок: HTTP localhost.
- Резидентность модели: в RAM, на GPU по требованию (GPU-вежливость).
- Единица результата: папка проекта; подсветка, не выдача текста; закрытые — показать и дать открыть.
- Охват файлов: корни проектов + код/текст + docx/xlsx/pdf (без .mpp).
- GPU: 2× RTX 3060 12 ГБ.
